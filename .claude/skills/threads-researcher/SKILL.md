# Threads Researcher

Threads上のバズ投稿を検索・収集し、競合アカウントの分析を行うリサーチエージェント。

## いつ使うか

- ユーザーが「リサーチ」「競合分析」「バズ投稿を探して」「トレンド」と言ったとき
- 投稿生成の前に市場調査が必要なとき
- 新しいテーマやジャンルの投稿傾向を調べたいとき

## ワークフロー

### Step 1: 対象アカウントとキーワードを特定

ユーザーに確認するか、アカウントのナレッジから自動判定する。

- **ポンタの場合**: `ポンタ/ナレッジ/03_genre.md` を読んでジャンルを把握
- **ルナの場合**: `ルナ/ナレッジ/03_genre.md` を読んでジャンルを把握

デフォルトの検索キーワード:
- ポンタ: 「AI 活用」「AI 副業」「借金返済」
- ルナ: 「星座占い」「恋愛運」「タロット 恋愛」

### Step 2: Threads APIでキーワード検索を実行

`threads_api.py` の `search_threads()` 関数を使用する。

```bash
cd "合同会社Harbor/Threads運用"
source ~/.zshrc
python3 -c "
from threads_api import *
get_user_profile('アカウントキー')
results = search_threads('アカウントキー', 'キーワード', limit=10)
"
```

複数キーワードで検索し、結果を統合する。

### Step 3: バズ投稿の分析

収集した投稿を以下の観点で分析する:

1. **表示回数TOP5の投稿を抽出**
2. **各投稿のバズ要因を分析**:
   - 使われているテクニック（`共通/ナレッジ/05_writing.md` の6つのバズテクニックに照らし合わせ）
   - フックの構造
   - CTAの有無と種類
   - 文字数、絵文字の使い方
   - ターゲットの明確さ
3. **共通パターンの抽出**: バズっている投稿に共通する要素

### Step 4: レポート出力

以下のフォーマットでレポートを出力する:

```markdown
# リサーチレポート - {アカウント名} ({日付})

## 検索キーワード
- キーワード1: XX件
- キーワード2: XX件

## バズ投稿TOP5
### 1位: @username | XX,XXX表示 | XXいいね
> 投稿テキスト（冒頭150文字）

**バズ要因**: 一次情報×再現性のコンビ。冒頭で具体的数字を提示...

### 2位: ...

## 共通パターン
- パターン1: ...
- パターン2: ...

## 今日の投稿への活用ポイント
1. ...
2. ...
3. ...
```

### Step 5: リサーチ結果の保存

レポートを `{アカウント名}/リサーチ/research_{日付}.md` に保存する。
また、特に参考になるバズ投稿は `{アカウント名}/ナレッジ/06_references.md` に追記する。

## 注意事項

- API制限に注意: 検索は1キーワードずつ、1秒以上間隔を空ける
- NGルール（`共通/ナレッジ/07_ng-rules.md`）に抵触する投稿は参考にしない
- リサーチ結果はDiscord通知（`discord_notify.py`）で分析スレッドに送信可能
